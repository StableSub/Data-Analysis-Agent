# AI Agent 개요

## 문서 목적

이 문서는 현재 프로젝트에서 AI Agent가 맡는 역할과 책임을 설명한다.
현재 구현 기준의 Agent 구조를 바탕으로 정리하며, 기획, 프론트엔드, 백엔드가 함께 참고할 수 있도록 쉬운 언어로 설명한다.

## AI Agent란 무엇인가

이 프로젝트에서 AI Agent는 단순히 한 번의 LLM 호출로 답변을 만드는 기능이 아니다.
사용자 질문을 받아 어떤 처리 경로로 보낼지 판단하고, 필요한 단계를 연결하고, 중간 승인과 재개를 포함한 실행 흐름 전체를 관리하는 시스템에 가깝다.

즉, AI Agent는 “질문을 받아 적절한 처리 경로를 정하고, 여러 도메인 기능을 연결해 최종 결과를 만드는 실행 계층”이라고 볼 수 있다.

## AI Agent의 핵심 역할

현재 AI Agent가 맡는 핵심 역할은 아래와 같다.

- 질문의 성격을 이해하고 처리 경로를 정한다.
- 데이터셋과 지침서 같은 컨텍스트를 활용해 판단 근거를 만든다.
- 필요 시 clarification, preprocess, analysis, rag, visualization, report 흐름으로 연결한다.
- 전처리, 시각화, 리포트처럼 사용자 승인이 필요한 단계를 관리한다.
- 실행 결과를 종합해 사용자에게 적절한 결과 유형으로 반환한다.

중요한 점은, AI Agent가 직접 모든 기능을 수행하는 것이 아니라 필요한 모듈들을 조합해 하나의 실행 흐름으로 만든다는 것이다.

## 일반 질문과 데이터 기반 질문에서의 역할 차이

AI Agent의 첫 번째 중요한 역할은 질문의 종류를 구분하는 것이다.

데이터셋이 없는 일반 질문은 일반 답변 경로로 처리된다.
이 경우에는 복잡한 데이터 준비나 분석 흐름 없이, 질문을 이해하고 바로 일반 응답을 만드는 것이 중심이 된다.

반대로 데이터셋이 있는 질문은 데이터 기반 흐름으로 들어간다.
이때는 dataset context, guideline context, planner 판단 같은 준비 단계를 거쳐야 하며, 이후 전처리, 분석, 설명형 질의응답, 시각화, 리포트 같은 경로가 상황에 따라 달라진다.

즉, AI Agent는 질문을 모두 같은 방식으로 처리하지 않고, 데이터 유무와 질문 성격에 따라 다른 실행 전략을 선택한다.

## AI Agent의 주요 구성 요소

현재 구현 기준으로 AI Agent는 하나의 모델이나 하나의 서비스로 이루어져 있지 않다.
주요 구성 요소는 아래와 같다.

### 1. AgentClient

AgentClient는 AI Agent 실행의 진입점 역할을 한다.
질문을 workflow 실행으로 넘기고, 진행 상태를 스트리밍으로 전달하고, 승인 이후 resume 실행을 연결한다.

### 2. Orchestration 계층

Orchestration은 전체 workflow를 조합하는 중심 계층이다.
질문 분기, planner 판단, 전처리, 분석, RAG, 시각화, 리포트 흐름을 연결해 하나의 실행 경로를 만든다.

### 3. Planner

Planner는 어떤 경로로 처리할지 판단하는 역할을 맡는다.
일반 질문인지, 분석이 필요한지, 설명형 질의응답으로 가야 하는지, 전처리/시각화/리포트가 필요한지를 결정한다.

### 4. Domain Workflows

실제 실행 기능은 각 도메인 workflow가 담당한다.
현재 기준으로는 preprocess, analysis, rag, guideline, visualization, report가 주요 workflow에 해당한다.

### 5. State / State View

AI Agent는 실행 중 상태를 공유하면서 동작한다.
질문, planning 결과, 전처리 결과, 분석 결과, 시각화 결과, 리포트 결과 같은 정보가 상태로 누적되며, 이 상태를 바탕으로 다음 단계와 최종 응답이 결정된다.

이 섹션에서 중요한 점은, AI Agent가 “하나의 모델”이 아니라 “여러 구성 요소가 함께 움직이는 workflow 기반 시스템”이라는 사실이다.

## 다른 백엔드 모듈과의 관계

AI Agent는 백엔드 안에서 모든 기능을 직접 구현하는 계층이 아니다.
도메인 모듈을 호출하고 순서를 결정하는 연결 계층에 가깝다.

- `chat` 모듈은 사용자 질문을 AI Agent 실행으로 연결하는 진입점이다.
- `datasets` 와 `profiling`은 데이터 기반 판단에 필요한 입력을 제공한다.
- `analysis`, `rag`, `visualization`, `reports`는 Agent가 상황에 맞게 호출하는 기능 모듈이다.
- `guidelines` 는 지침서 기반 판단에 필요한 내부 지식 입력을 제공한다.

즉, AI Agent는 기능 자체보다 실행 순서와 연결 구조를 책임지는 계층이며, 이것이 일반적인 백엔드 서비스 모듈과 다른 점이다.

## 사용자 경험에서 AI Agent가 보이는 지점

AI Agent는 백엔드 내부 계층이지만, 사용자 경험에서도 분명히 드러난다.

- 질문을 보내면 진행 상태가 thought 또는 streaming 형태로 보인다.
- 전처리, 시각화, 리포트 단계에서는 승인 요청이 발생할 수 있다.
- 질문이 모호하면 clarification 질문이 다시 올 수 있다.
- 최종 결과는 일반 답변, 데이터 분석 요약, 시각화 결과, 리포트 등 여러 형태로 나타날 수 있다.

즉, AI Agent는 내부 실행 로직이면서 동시에 사용자가 체감하는 대화형 분석 경험의 핵심이다.

## 이 문서를 읽는 방법

이 문서는 AI Agent의 역할과 책임을 설명하는 개요 문서다.
질문이 어떤 순서로 실행되는지 보고 싶다면 `시스템 플로우 개요`와 이후 `AI Agent 실행 흐름` 문서를 보는 것이 맞다.

관련 문서는 아래 순서로 이어서 읽는 것이 자연스럽다.

- 시스템 플로우 개요: 현재 구현 기준의 전체 분기 흐름
- 시스템 아키텍처: 시스템 전체 구성 요소와 연결 관계
- 백엔드 구조: 백엔드 모듈과 오케스트레이션 구조
- AI Agent 실행 흐름: AI Agent가 실제로 어떤 단계로 실행되는지

# 문제 정의 및 제품 방향

## 문서 목적

이 문서는 우리 제품이 해결하려는 문제와 제품의 방향을 제품 관점에서 설명한다.
현재 기준의 근거 자료는 산학캡스톤 프로젝트 제안서이며, 이 문서는 제안서 내용을 바탕으로 문제, 필요성, 가치, 방향을 재구성한 초안이다.

## 해결하려는 문제

기업과 조직은 이미 다양한 데이터를 수집하고 있지만, 실제 현장에서는 그 데이터를 분석해 의미 있는 인사이트로 전환하는 과정이 여전히 전문가 중심으로 이루어지고 있다.
데이터가 존재하는 것과 데이터를 활용해 의사결정에 반영하는 것은 별개의 문제이며, 이 간극이 현재 해결하려는 핵심 문제다.

## 현재 방식의 한계

현재의 데이터 분석 업무는 분석 도구 사용 경험, 데이터 해석 능력, 시각화와 리포트 작성 역량을 갖춘 사람에게 크게 의존한다.
이 때문에 비전문가는 데이터를 직접 활용하기 어렵고, 단순한 질문조차 분석 요청과 해석 과정을 별도로 거쳐야 하는 경우가 많다.

또한 분석 과정이 수작업 중심으로 진행되면 질문을 이해하고, 데이터를 살펴보고, 필요한 결과를 시각화하고, 이를 다시 설명 가능한 형태로 정리하는 데 시간과 비용이 반복적으로 들어간다.

## 왜 지금 이 제품이 필요한가

제안서 기준으로, 최근 ChatGPT, Claude, Gemini와 같은 생성형 AI 기술이 빠르게 발전하면서 자연어를 기반으로 데이터를 해석하고 분석을 보조하는 방식이 현실적인 구현 대상으로 바뀌고 있다.
이는 기존의 수작업 기반 분석 업무를 자동화하고, 더 많은 사용자가 데이터에 접근할 수 있도록 돕는 방향으로의 전환 필요성을 만든다.

즉, 지금은 데이터를 많이 보유한 조직이 더 쉽게 데이터를 해석하고 활용할 수 있도록, 생성형 AI를 활용한 분석 도우미를 제품 형태로 구체화할 시점이다.

## 제품이 제공하려는 핵심 가치

이 제품은 사용자가 자연어로 분석을 요청하고, 시스템이 그 요청을 바탕으로 데이터 분석을 수행할 수 있도록 돕는 것을 핵심 가치로 둔다.

제안서 기준의 핵심 가치는 아래와 같다.

- 자연어 기반 데이터 분석 요청
- 분석 과정의 자동화 및 보조
- 자동 시각화 생성
- 인사이트 리포트 생성

## 주요 사용자와 사용 맥락

현재 제안서에는 구체적인 사용자군이 명시되어 있지는 않다.
따라서 첫 버전에서는 아래와 같은 수준으로 사용자와 사용 맥락을 정의한다.

- 데이터 분석 비전문가이지만 업무상 데이터를 이해해야 하는 사용자
- 실무 데이터를 바탕으로 빠르게 인사이트를 확인하고 싶은 사용자
- 분석 결과를 시각화와 리포트 형태로 정리할 필요가 있는 조직 사용자

이 제품은 복잡한 분석 도구를 직접 다루기보다, 대화형 인터페이스를 통해 질문하고 결과를 확인하는 사용 맥락에 적합한 방향으로 설계된다.

## 제품 방향

제품 방향은 `생성형 AI 기반 Data Analysis AI Agent`를 중심으로 한다.
사용자는 웹 기반 대화형 인터페이스에서 자연어로 질문하고, 시스템은 데이터를 해석하고 분석하며, 필요 시 시각화와 리포트까지 연결되는 경험을 제공하는 것을 목표로 한다.

즉, 이 제품은 단순한 챗봇이 아니라 데이터 분석 요청을 이해하고, 결과를 설명 가능한 형태로 반환하는 분석 보조 시스템을 지향한다.

## 초기 구현 범위

제안서의 개발 목표를 기준으로, 초기 구현 범위는 아래 네 가지로 정리한다.

1. Data Analysis AI Agent 설계
2. 생성형 AI 모델 연동
3. 자동 시각화 및 인사이트 리포트 생성
4. 웹 기반 대화형 UI 개발

이 문서에서는 위 항목을 제품의 초기 범위로만 정의하며, 세부 기능 단위의 요구사항은 별도 PRD 문서에서 다룬다.

## 기대 효과

이 제품이 구현되면 데이터 분석 과정에서 전문가 의존도를 일부 완화하고, 비전문가의 데이터 접근성을 높이는 데 도움이 될 수 있다.
또한 자연어 요청, 자동 분석 보조, 시각화, 리포트 생성을 하나의 흐름으로 연결함으로써 분석 업무의 효율을 높이는 방향을 기대할 수 있다.

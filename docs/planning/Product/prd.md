# 제품 요구사항 문서(PRD)

## 문서 목적

이 문서는 제품 요구사항과 범위를 정리한다.
현재 기준에서는 산학캡스톤 프로젝트 제안서, 문제 정의 및 제품 방향 문서, 대표 사용자 시나리오, 사용자 경험 설계 문서를 바탕으로 초기 버전의 제품 요구사항을 정리한다.

## 제품 목표

이 제품의 목표는 사용자가 자연어로 데이터를 분석 요청하면, 시스템이 데이터 확인, 전처리 제안, 분석 수행, 결과 요약까지 하나의 흐름으로 지원하는 `생성형 AI 기반 Data Analysis AI Agent`를 제공하는 것이다.

초기 버전에서는 특히 데이터 분석 비전문가도 분석 대상 데이터를 이해하고, 이상 구간과 원인 후보를 확인하고, 결과를 활용 가능한 형태로 받아볼 수 있도록 하는 데 초점을 둔다.

## 해결하려는 문제 요약

현재 데이터 분석 업무는 여전히 전문가와 수작업에 크게 의존하고 있다.
이 때문에 비전문가는 데이터를 직접 해석하기 어렵고, 단순한 분석 요청도 별도의 분석 과정과 결과 정리 단계를 거쳐야 한다.

이 제품은 이러한 문제를 줄이기 위해 자연어 기반 분석 요청과 자동화된 분석 보조 경험을 제공하고자 한다.

## 핵심 사용자

초기 버전의 핵심 사용자는 아래와 같이 정의한다.

- 데이터 분석 비전문가이지만 업무상 데이터를 이해해야 하는 사용자
- 생산 또는 운영 데이터를 바탕으로 상태를 빠르게 진단하고 싶은 사용자
- 분석 결과를 시각화와 요약 형태로 확인하고 후속 조치에 활용하려는 사용자

## 핵심 사용자 흐름

제품은 아래 흐름을 기본 사용자 흐름으로 지원해야 한다.

1. 사용자가 분석 대상 데이터를 선택한다.
2. 사용자가 자연어로 분석 요청을 입력한다.
3. 시스템이 데이터를 조회하고 주요 상태를 확인할 수 있게 보여준다.
4. 필요 시 시스템이 전처리를 제안하고, 사용자가 승인 여부를 결정한다.
5. 시스템이 분석을 수행하고 이상 구간, 핵심 변수, 원인 후보를 제시한다.
6. 시스템이 최종 결과를 요약하고 후속 조치에 활용 가능한 형태로 정리한다.

## 핵심 기능 요구사항

### 1. 데이터 선택 및 분석 대상 지정

- 사용자는 분석 대상이 되는 데이터셋 또는 분석 구간을 지정할 수 있어야 한다.
- 시스템은 사용자가 지정한 범위를 기준으로 분석을 시작할 수 있어야 한다.

### 2. 자연어 기반 분석 요청

- 사용자는 복잡한 분석 도구 조작 없이 자연어로 질문하거나 진단 요청을 할 수 있어야 한다.
- 시스템은 사용자의 요청 의도를 이해하고, 데이터 분석 흐름으로 연결할 수 있어야 한다.

### 3. 초기 데이터 확인

- 시스템은 분석 시작 전 주요 변수의 추이와 데이터 상태를 확인할 수 있는 정보를 제공해야 한다.
- 사용자는 현재 어떤 데이터가 분석 대상인지, 어떤 변수들이 중요한지 이해할 수 있어야 한다.

### 4. 전처리 제안 및 승인

- 시스템은 분석 정확도 향상을 위해 필요한 경우 전처리를 제안할 수 있어야 한다.
- 전처리 제안 시, 왜 필요한지와 어떤 변화가 생기는지 사용자가 이해할 수 있어야 한다.
- 전처리는 사용자 승인 후에만 진행되어야 한다.

### 5. 분석 수행

- 시스템은 선택된 데이터에 대해 이상 탐지 또는 상태 진단 분석을 수행할 수 있어야 한다.
- 분석 결과는 단순 수치 나열이 아니라 이상 여부, 이상 구간, 주요 원인 후보가 드러나는 형태여야 한다.

### 6. 원인 분석 보조

- 시스템은 분석 결과와 연결된 핵심 변수 또는 이상 징후를 바탕으로 원인 후보를 제시할 수 있어야 한다.
- 필요 시 관련 지식이나 매뉴얼 기반 설명을 함께 제공할 수 있어야 한다.

### 7. 시각화 및 결과 요약

- 시스템은 분석 결과를 사용자가 빠르게 이해할 수 있는 시각적 형태로 제공할 수 있어야 한다.
- 최종 결과는 현재 상태, 주요 이상 구간, 핵심 원인 후보, 우선 조치 방향이 요약된 형태로 정리되어야 한다.

### 8. 웹 기반 대화형 인터페이스

- 전체 제품 경험은 웹 기반 인터페이스에서 제공되어야 한다.
- 사용자는 질문 입력, 승인, 결과 확인을 하나의 흐름에서 수행할 수 있어야 한다.

## 우선순위

### Must Have

- 데이터 선택 또는 분석 범위 지정
- 자연어 기반 분석 요청
- 전처리 제안과 승인
- 이상 탐지 또는 상태 진단 분석
- 결과 요약 제공
- 웹 기반 대화형 인터페이스

### Should Have

- 전처리 전후 비교 제공
- 주요 변수 시각화
- 원인 후보 설명
- 후속 조치 방향 제시

### Could Have

- 관련 지식이나 매뉴얼과 연결된 보조 설명
- 결과 공유 또는 전달을 위한 보조 기능

## 제외 범위

초기 버전에서는 아래 항목을 필수 범위에 포함하지 않는다.

- 다양한 산업군을 모두 포괄하는 범용 분석 시나리오
- 복잡한 분석 파이프라인을 사용자가 직접 세밀하게 조정하는 기능
- 정교한 권한 관리나 협업 워크플로우
- 고도화된 외부 알림, 메일, 메신저 연동 기능
- 경쟁 제품 수준의 완성형 리포트 디자인 시스템

## 성공 기준

초기 버전의 성공 기준은 아래와 같이 정의한다.

- 사용자가 자연어로 분석 요청을 시작할 수 있다.
- 시스템이 분석 대상 데이터를 기준으로 상태 진단 흐름을 수행할 수 있다.
- 전처리 필요 시 사용자 승인 기반으로 흐름이 이어진다.
- 사용자가 이상 구간, 주요 변수, 원인 후보, 후속 조치 방향을 결과에서 이해할 수 있다.
- 전체 경험이 웹 기반 인터페이스 안에서 하나의 흐름으로 연결된다.

## 향후 확장 가능 항목

향후에는 아래와 같은 방향으로 확장할 수 있다.

- 더 다양한 분석 시나리오 지원
- 도메인 지식 연계 강화
- 리포트 품질 고도화
- 협업 및 공유 기능 확장

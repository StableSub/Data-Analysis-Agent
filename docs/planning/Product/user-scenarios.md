# 대표 사용자 시나리오

## 목적

이 문서는 대표 사용 사례를 시나리오 중심으로 정리한다.

## 대표 시나리오 1. 사출성형 데이터 이상 진단

### 상황

사용자는 사출성형 설비에서 수집된 생산 데이터를 기준으로 특정 구간의 설비 상태를 진단하고자 한다.
분석 대상은 최근 생산된 Lot 데이터이며, `No_Shot 1000~2000` 구간을 중심으로 상태 이상 여부와 원인 후보를 확인하는 것이 목적이다.

### 사용자 요청

사용자는 웹 대시보드에서 진단 기능을 실행한 뒤, 최근 생산 구간을 기준으로 설비 상태를 진단해 달라고 요청한다.

### 시스템 진행 단계

#### 1. 데이터 조회와 초기 탐색

시스템은 사용자가 지정한 Shot 구간의 데이터를 불러온다.
이후 주요 공정 변수(Cycle Time, Injection Pressure, Barrel Temperature 등)의 변화 추이를 시계열 차트로 보여주고, 분석 대상 구간의 전반적인 상태를 확인할 수 있게 한다.

동시에 전처리 전 데이터 분포를 함께 제시하여, 변수별 스케일 차이와 분포 특성을 사용자가 확인할 수 있도록 한다.

#### 2. 전처리 필요성 제안

시스템은 분석 정확도를 높이기 위해 필요한 전처리 방식을 제안한다.
이 시나리오에서는 오토인코더 기반 이상 탐지를 위해 변수 스케일을 맞추는 표준화 전처리가 필요하다고 안내한다.

사용자는 이 제안을 검토한 뒤 전처리 진행 여부를 결정한다.

#### 3. 사용자 승인 후 전처리 실행

사용자가 전처리에 동의하면 시스템은 전처리를 수행하고, 전처리 결과를 시각적으로 비교할 수 있는 형태로 보여준다.
이를 통해 사용자는 승인 이전과 이후의 데이터 상태 차이를 확인할 수 있다.

#### 4. 이상 탐지 수행

전처리된 데이터는 이상 탐지 모델에 입력된다.
시스템은 Shot 단위로 이상 점수를 계산하고, 정상 범위를 벗어나는 구간이 있는지 확인한다.

이상 구간이 발견되면 해당 구간을 강조하여 보여주고, 어떤 변수가 이상 징후와 가장 강하게 연결되는지 함께 제시한다.
이 시나리오에서는 `No_Shot 1850~1870` 구간이 이상 구간으로 식별되고, `Injection Pressure`가 주요 원인 후보로 제시된다.

#### 5. 원인 분석 보조

시스템은 이상 징후와 연결된 핵심 변수를 바탕으로 관련 지식과 매뉴얼을 검색한다.
이를 통해 사용자는 설비 이상과 관련된 원인 후보와 점검 방향을 함께 확인할 수 있다.

예를 들어, 사출 압력 저하와 연관된 부품 마모나 노즐 막힘 가능성을 근거 자료와 함께 제시할 수 있다.

#### 6. 결과 정리와 후속 조치 안내

시스템은 분석 결과를 종합하여 설비 상태, 주요 이상 구간, 원인 후보, 우선 확인 항목을 한 번에 볼 수 있는 형태로 정리한다.
사용자는 이를 바탕으로 유지보수 팀에 전달하거나 후속 점검을 진행할 수 있다.

### 사용자가 받는 최종 결과

이 시나리오에서 사용자는 아래 결과를 받는다.

- 지정한 생산 구간의 주요 공정 변수 추이
- 전처리 필요성 안내와 전처리 결과 비교
- 이상 구간 탐지 결과
- 핵심 원인 후보 변수
- 관련 지식 검색 결과와 점검 방향
- 최종 요약 결과와 후속 조치 정보

### 이 시나리오의 의미

이 시나리오는 사용자가 단순히 차트를 보는 수준을 넘어, 자연어 요청만으로 데이터 확인, 전처리 승인, 이상 탐지, 원인 분석, 결과 정리까지 하나의 흐름으로 지원받는 사용 경험을 보여준다.

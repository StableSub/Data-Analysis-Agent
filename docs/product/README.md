# 제품 문서

이 디렉터리는 데이터 분석 AI 에이전트의 제품 목표, 현재 구현 기준, 개선 방향을 설명한다.
구현 상세보다 “무엇을 만들고 있으며, 현재 어디까지 되어 있고, 다음 우선순위가 무엇인지”를 판단하는 기준을 소유한다.

## 포함 문서

| 문서 | 역할 | 언제 읽는가 |
|---|---|---|
| [제품 요구사항 문서(PRD)](./prd.md) | 문제 정의, 사용자 흐름, 기능 요구사항, PoC 기준 | 제품 범위나 사용자 가치 판단이 필요할 때 |
| [현재 구현 기준선](./current-state-baseline.md) | 현재 repository가 실제로 제공하는 기능과 제약 | 현재 동작과 문서/코드 정합성을 확인할 때 |
| [구현 로드맵](./roadmap.md) | 개선 축, 단계별 우선순위, 하네스 우선순위 | 다음 작업의 우선순위를 정할 때 |

## 읽는 순서

1. 제품 방향을 처음 파악할 때는 [제품 요구사항 문서(PRD)](./prd.md)를 읽는다.
2. 현재 구현과 제약을 확인할 때는 [현재 구현 기준선](./current-state-baseline.md)을 읽는다.
3. 개선 작업의 우선순위나 검증 강화 방향은 [구현 로드맵](./roadmap.md)을 따른다.

## 유지 기준

- 제품 범위, 사용자 흐름, PoC 기준이 바뀌면 `prd.md`를 갱신한다.
- 실제 구현 상태가 바뀌면 `current-state-baseline.md`를 갱신한다.
- 개선 우선순위나 하네스 전략이 바뀌면 `roadmap.md`를 갱신한다.
- architecture 문서와 제품 문서가 다르면, 현재 runtime 설명은 architecture 문서와 실제 코드를 우선한다.

# 리뷰 및 검증 운영 가이드

## 목적

캡스톤 발표 전까지 팀이 PR 리뷰와 데모 검증을 같은 기준으로 수행하도록 체크리스트를 정리한다.
이 문서는 코드 스타일 규칙보다 **승인 전에 무엇을 확인해야 하는가**에 집중한다.

## 리뷰 역할 분리

- **작성자**: 변경 이유, 범위, 검증 결과를 정리한다.
- **영역 리뷰어**: 해당 영역 코드/문서가 현재 구조와 맞는지 확인한다.
- **발표 관점 리뷰어**: 이 변경을 발표에서 어떻게 설명할지, 리스크를 숨기지 않았는지 확인한다.

한 사람이 여러 역할을 맡을 수 있지만, 제출 체크와 승인 체크는 분리해서 본다.

## PR 리뷰 순서

1. 변경 범위 확인
2. 소유 영역 일치 여부 확인
3. 검증 명령과 결과 확인
4. 사용자/데모 흐름 영향 확인
5. 발표 메시지와 리스크 문장 확인

## 작성자 체크리스트

- [ ] 제목이 변경 의도를 설명한다
- [ ] 관련 파일/문서 경로를 PR 본문에 적었다
- [ ] 실행한 명령과 결과를 적었다
- [ ] 데모에 보이는 변화가 있으면 스크린샷 또는 설명을 남겼다
- [ ] 아직 자동화되지 않은 검증은 수동 확인이라고 명시했다

## 리뷰어 체크리스트

### 공통

- [ ] 변경이 요청 범위를 벗어나지 않는다
- [ ] 삭제/수정된 설명이 현재 코드와 맞다
- [ ] 검증 명령이 저장소에 실제 존재한다
- [ ] 오류 처리, 승인 흐름, 발표 경로에 대한 언급이 과장되지 않았다

### 문서 변경

- [ ] 코드 경로가 실제로 존재한다
- [ ] architecture 문서는 runtime 동작을 설명한다
- [ ] 팀 문서는 AGENTS 대체 문서처럼 행동하지 않는다

### 프론트엔드 변경

- [ ] `npm --prefix frontend run build` 결과가 있다
- [ ] SSE, approval, error UI 영향이 있으면 Workbench 수동 확인이 있다

### 백엔드 변경

- [ ] 관련 pytest 결과가 있다
- [ ] public API/SSE 계약이 바뀌면 문서가 함께 갱신되었다

## 검증 결과 기록 형식

권장 형식:

```md
## Verification
- PASS `PYTHONPATH=. pytest -q backend/tests/test_architecture_docs.py backend/tests/test_docs_harness.py`
- PASS `npm --prefix frontend run build`
- MANUAL Workbench에서 dataset 선택 → 질문 → approval_required → resume 확인
- HOLD benchmark 자동화 없음, 발표용 수동 측정 필요
```

## 반려 기준

다음 중 하나라도 해당하면 승인하지 않는다.

- 검증 없이 “문제없음”만 적은 PR
- 없는 lint/typecheck 체계를 근거로 든 PR
- 발표 데모 핵심 흐름을 건드렸지만 담당자/검증자가 없는 PR
- 문서가 미래 계획을 현재 구현처럼 서술하는 경우

## 발표 직전 verification runbook

### 1. 기술 검증

- [ ] backend 문서 테스트 통과
- [ ] 핵심 workflow 회귀 테스트 통과
- [ ] frontend build 통과

### 2. 데모 리허설

- [ ] dataset 업로드
- [ ] 질문 실행
- [ ] approval_required 노출
- [ ] revise/approve 후 resume
- [ ] 최종 answer/visualization/report 확인
- [ ] 오류 발생 시 메시지와 복구 경로 확인

### 3. 발표 메시지 검증

- [ ] “현재 가능한 것”과 “향후 계획”을 구분했다
- [ ] benchmark 부재/제한사항을 숨기지 않았다
- [ ] trace, approval, evidence 흐름을 한 문장으로 설명할 수 있다

## 발표용 framing

리뷰의 목적은 결함을 찾는 것만이 아니라, **발표에서 방어 가능한 상태로 만드는 것**이다.
따라서 리뷰 코멘트는 “이게 왜 틀렸는가”와 함께 “발표에서 어떻게 설명해야 하는가”까지 남긴다.

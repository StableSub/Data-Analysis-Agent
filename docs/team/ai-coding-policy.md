# AI 코딩 정책

## 목적

이 문서는 캡스톤 발표 전까지 이 저장소에서 AI 도구를 사용할 때의 최소 운영 규칙을 정리한다.
`AGENTS.md`처럼 에이전트를 제어하는 시스템 프롬프트가 아니라, 사람이 합의해서 PR·문서·발표 품질을 지키기 위한 팀 정책이다.

## 적용 범위

- 코드 작성·수정·리팩터링에 AI를 사용할 때
- 문서 초안, 테스트 정리, 발표 자료 근거 정리에 AI를 사용할 때
- Codex/ChatGPT/Claude/Gemini 등 도구 종류와 무관하게 동일하게 적용한다

## 기본 원칙

1. **AI 출력은 초안이다.** 최종 책임은 작성자와 리뷰어에게 있다.
2. **코드와 문서가 다르면 코드를 기준으로 수정한다.** 추측성 설명을 금지한다.
3. **존재하지 않는 검증을 통과했다고 주장하지 않는다.** 현재 `frontend/package.json`에는 `dev`, `build`만 있고 `lint`, `check:types` 스크립트는 없다.
4. **작은 diff를 우선한다.** 한 PR에 기능 추가, 구조 변경, 문서 정리를 섞지 않는다.
5. **근거를 남긴다.** AI가 제안한 변경이라도 관련 파일 경로와 실제 실행한 검증 명령을 PR 본문에 남긴다.
6. **민감 정보는 입력하지 않는다.** 실제 사용자 데이터, 비밀키, 발표 전 비공개 자료를 외부 AI에 그대로 붙여넣지 않는다.

## AI 사용 허용 패턴

### 권장

- 기존 코드 구조 설명 요청
- 반복적인 문서 초안 작성
- 테스트 케이스 후보 정리
- API 계약 요약
- 발표 슬라이드용 핵심 메시지 정리

### 금지 또는 주의

- 실행하지 않은 테스트 결과를 사실처럼 서술
- 없는 파일·명령·모듈을 전제로 한 변경
- 대규모 자동 치환 후 무검증 커밋
- backend/runtime 동작을 확인하지 않은 architecture 문서 수정
- 팀 합의 없이 새 dependency 추가 제안 수용

## 저장소 기준 체크포인트

- 백엔드 public API 기준: `backend/app/main.py`, `backend/app/modules/*/router.py`
- SSE/approval/resume 계약 기준: `backend/app/modules/chat/router.py`, `backend/app/modules/chat/service.py`, `backend/app/orchestration/client.py`, `frontend/src/app/hooks/useAnalysisPipeline.ts`
- 문서 검증 기준: `backend/tests/test_architecture_docs.py`, `backend/tests/test_docs_harness.py`
- 프론트엔드 검증 기준: `npm --prefix frontend run build`

## 작업 전 체크리스트

- [ ] 변경 대상 파일과 소유 범위를 명확히 적었다
- [ ] 현재 코드 또는 문서를 먼저 읽고 사실 관계를 확인했다
- [ ] 이번 작업이 문서-only인지 코드 변경인지 구분했다
- [ ] 필요한 검증 명령을 작업 전에 적어두었다

## 작업 후 체크리스트

- [ ] AI가 만든 표현 중 과장·추측 문장을 제거했다
- [ ] 실제 실행한 검증 명령과 결과를 기록했다
- [ ] 문서 변경이면 코드 경로 참조가 살아 있는지 확인했다
- [ ] 발표에서 바로 말할 수 있는 한 줄 요약을 정리했다

## PR 본문 권장 템플릿

```md
## 변경 이유
- 왜 이 변경이 필요한가

## AI 사용 범위
- 초안 작성 / 코드 제안 / 테스트 아이디어 / 문장 다듬기

## 사람이 직접 확인한 항목
- 코드 경로 확인:
- 수동 검토 포인트:
- 실행한 검증:

## 발표 관점 메시지
- 이 변경이 신뢰성, 정확성, 재현성 중 무엇을 강화하는가
```

## 발표용 framing

캡스톤 발표에서는 “AI를 썼다”보다 **AI를 통제했다**는 메시지가 중요하다.
따라서 팀은 다음 세 문장을 유지한다.

- AI 출력은 항상 사람 검토 후 반영했다.
- 검증 가능한 명령과 코드 경로를 근거로 설명한다.
- 없는 성능 지표나 자동화를 있다고 포장하지 않는다.

# 팀 문서

이 디렉터리는 팀원이 AI 에이전트와 협업할 때 필요한 운영 규칙, 역할 분담, 리뷰와 검증 기준을 모아둔다.
루트 `AGENTS.md`가 시스템에게 주는 실행 지침이라면, 이 디렉터리는 사람이 작업을 요청하고 검토할 때 참고하는 팀 운영 문서다.

## 포함 문서

| 문서 | 역할 | 언제 읽는가 |
|---|---|---|
| [AI 코딩 정책](./ai-coding-policy.md) | AI를 활용한 코드 작성 원칙과 금지 패턴 | 작업 시작 전 공통 기준이 필요할 때 |
| [역할과 소유권](./roles-and-ownership.md) | 팀 역할, 문서/코드 소유권, 책임 경계 | 담당자를 정하거나 변경 영향 범위를 나눌 때 |
| [리뷰와 검증](./review-and-verification.md) | 리뷰 관점, 검증 루프, 완료 기준 | PR/작업 완료 전 점검할 때 |
| [에이전트 프롬프트 플레이북](./agent-prompt-playbook.md) | AI에게 일을 맡기는 프롬프트 패턴 | 재작업을 줄이는 요청문을 작성할 때 |

## 읽는 순서

1. 팀 공통 원칙은 [AI 코딩 정책](./ai-coding-policy.md)을 먼저 읽는다.
2. 담당 영역과 책임 경계는 [역할과 소유권](./roles-and-ownership.md)을 확인한다.
3. 작업 완료 전에는 [리뷰와 검증](./review-and-verification.md)을 기준으로 점검한다.
4. AI에게 작업을 맡길 때는 [에이전트 프롬프트 플레이북](./agent-prompt-playbook.md)의 템플릿을 활용한다.

## 유지 기준

- 팀의 작업 방식이나 리뷰 기준이 바뀌면 이 디렉터리의 문서를 함께 갱신한다.
- 시스템 실행 규칙이 바뀌면 루트 `AGENTS.md`와 충돌하지 않게 정리한다.
- 실제 검증 명령이 바뀌면 `docs/development/`와 `docs/testing/`도 함께 확인한다.

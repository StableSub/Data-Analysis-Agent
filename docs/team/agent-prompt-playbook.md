# 에이전트 프롬프트 플레이북

## 목적

이 문서는 이 저장소에서 AI 에이전트에게 일을 맡길 때 재작업을 줄이는 프롬프트 패턴을 정리한다.
인간 팀 문서이며, 시스템용 `AGENTS.md`를 대체하지 않는다.

## 좋은 프롬프트의 조건

1. **범위가 명확하다** — 어떤 파일을 바꿀지 지정한다.
2. **금지 범위가 있다** — 건드리면 안 되는 파일/기능을 적는다.
3. **검증 명령이 있다** — 무엇을 실행해 결과를 보여줄지 적는다.
4. **완료 기준이 있다** — 문서 갱신, 테스트 통과, 발표용 요약 등 산출물을 명시한다.

## 저장소 맞춤 템플릿

### 문서-only 작업

```text
다음 파일만 수정하세요: <paths>
코드는 수정하지 마세요.
현재 코드 기준으로 사실만 반영하세요.
존재하지 않는 lint/check-types/CI를 있다고 쓰지 마세요.
작업 후 아래 명령 결과를 보고하세요:
- PYTHONPATH=. pytest -q backend/tests/test_architecture_docs.py backend/tests/test_docs_harness.py
```

### 프론트엔드 작업

```text
변경 범위: <frontend paths>
다른 디렉터리는 건드리지 마세요.
SSE/approval 흐름에 영향이 있으면 관련 코드 경로를 함께 검토하세요.
작업 후 아래를 실행하세요:
- npm --prefix frontend run build
수동 확인 포인트:
- Workbench에서 session / approval_required / done / error 흐름
```

### 백엔드 API/SSE 작업

```text
변경 범위: <backend paths>
API 또는 SSE 계약이 바뀌면 docs/system 문서도 함께 갱신하세요.
작업 후 관련 pytest와 문서 테스트를 실행하세요.
최종 보고에는 변경 파일, 계약 변경점, 프론트 영향도를 포함하세요.
```

## 프롬프트에 꼭 넣을 정보

- 목표 1문장
- 수정 가능 파일
- 수정 금지 파일
- 관련 참고 코드 경로
- 필수 검증 명령
- 최종 보고 형식

## 나쁜 프롬프트 예시

- “전체적으로 정리해줘”
- “필요한 거 다 고쳐줘”
- “테스트도 적당히 돌려줘”
- “CI 통과하게 해줘” (실제 CI 구성이 확인되지 않았을 때)

## 좋은 프롬프트 예시

### 예시 1: SSE 문서 보강

```text
`docs/system/api-sse-error-contract.md`만 작성하세요.
기준 코드는 `backend/app/modules/chat/router.py`, `backend/app/modules/chat/service.py`, `frontend/src/app/hooks/useAnalysisPipeline.ts`입니다.
오류 payload는 현재 구현 기준으로만 설명하고, 구조화된 error code가 있다고 쓰지 마세요.
작업 후 문서 테스트 결과를 보고하세요.
```

### 예시 2: 발표 전 운영 문서 작성

```text
`docs/team/`와 `docs/testing/` 아래 지정된 파일만 수정하세요.
사람 팀 운영 문서로 작성하고 AGENTS 대체 문서처럼 쓰지 마세요.
PR 리뷰 게이트, 역할 분담, benchmark 측정 기준을 포함하세요.
존재하지 않는 lint/check-types/CI를 있다고 쓰지 마세요.
```

## 리뷰 요청 프롬프트 팁

리뷰만 시킬 때는 수정과 리뷰를 섞지 않는다.
예:

```text
코드는 수정하지 말고 아래 파일을 리뷰하세요.
위험도 순으로 이슈를 정리하고, 각 이슈에 근거 파일 경로를 붙이세요.
추측은 추측이라고 표시하세요.
```

## 발표 준비 프롬프트 팁

발표 자료용 요청에는 반드시 아래를 포함한다.

- 현재 구현 기준인지, 향후 계획인지 구분
- 수치가 있으면 출처(테스트셋/수동 측정/trace)
- 데모에서 직접 보이는 흐름인지 여부

## 발표용 framing

좋은 프롬프트는 더 화려한 답을 만들기보다 **검증 가능한 답**을 만든다.
이 저장소에서는 “파일 경로 + 명령 + 완료 기준”이 포함된 프롬프트가 가장 재작업이 적다.

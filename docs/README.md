# Docs

프로젝트 문서는 아래 카테고리 기준으로 정리한다.

- `Product`: 문제 정의, UX, PRD, 시나리오
- `System`: 시스템 흐름, 구조, API, 용어
- `AI-Agent`: AI Agent 역할, 실행 흐름, 프롬프트, 로깅
- `Data`: 데이터 구조와 데이터베이스
- `Infra-Ops`: 실행 환경, 배포, 운영, 장애 대응
- `Decision-Log`: 구조 결정 기록과 변경 이력

## 디렉터리 구조

```text
docs/
  Product/
  System/
  AI-Agent/
  Data/
  Infra-Ops/
  Decision-Log/
```

## 문서 제목 변경 기준

- 너무 짧거나 약어만 있는 제목은 의미가 바로 드러나게 확장한다.
- 영어/한글이 섞인 제목은 한글 중심으로 통일한다.
- `개요`, `구조`, `설계`, `전략`, `기록`처럼 문서 성격이 드러나는 표현을 우선 사용한다.

# 🎨 피그마 디자인 화면 가이드

## 📋 개요

피그마 디자인 작업을 위해 애플리케이션의 모든 화면 상태를 독립적인 컴포넌트로 만들었습니다.
각 화면은 실제 컴포넌트를 사용하므로 정확한 레이아웃과 스타일을 확인할 수 있습니다.

## 🆕 새로운 단순화 레이아웃

**2024.11.10 업데이트**: 좌측 네비게이션 + 대화창 구조로 단순화했습니다!

- ✅ 맨 위에 "새 대화" 버튼
- ✅ 5개 주요 기능만 (AI 챗봇, 시각화, 편집, 시뮬레이션, 감사로그)
- ✅ 데이터 관리는 대화창 안에서 처리
- ✅ SessionSidebar 제거

## 🚀 빠른 시작

### 1. 갤러리 모드로 모든 화면 보기

**App.tsx**를 다음과 같이 수정하세요:

```tsx
import { ScreenGallery } from "./components/figma-screens/ScreenGallery";

function App() {
  return <ScreenGallery />;
}

export default App;
```

그 다음 개발 서버를 실행:

```bash
npm run dev
```

브라우저에서 http://localhost:3000 을 열면 모든 화면을 카드 형태로 볼 수 있습니다.

### 2. 화면 사용 방법

1. **갤러리에서 화면 선택**
   - 원하는 화면 카드 클릭
   - 전체 화면으로 해당 상태 표시

2. **테마 전환**
   - 우측 상단 "🌙 Dark" / "☀️ Light" 버튼 클릭
   - 다크/라이트 모드 즉시 전환

3. **스크린샷 캡처**
   - Windows: `Win + Shift + S`
   - Mac: `Cmd + Shift + 5`
   - 또는 브라우저 개발자 도구의 디바이스 모드 스크린샷 기능

4. **반응형 확인**
   - F12로 개발자 도구 열기
   - 디바이스 모드 활성화 (Ctrl/Cmd + Shift + M)
   - 원하는 해상도 선택 (모바일/태블릿/데스크톱)

## 🎯 피그마 디자인 작업 플로우

### 1단계: 화면 캡처

```
1. npm run dev 실행
2. ScreenGallery에서 원하는 화면 선택
3. Light 모드로 스크린샷 캡처
4. Dark 모드로 전환하여 다시 캡처
5. 필요시 개발자 도구로 다양한 해상도 캡처
```

### 2단계: 피그마로 가져오기

```
1. 피그마에서 새 프레임 생성
   - Desktop: 1920×1080
   - Tablet: 768×1024
   - Mobile: 375×812

2. 캡처한 이미지를 프레임에 배치

3. 이미지를 참고하여 디자인 작업
   - 색상 추출 (스포이드 도구)
   - 간격 측정 (Alt/Option 키)
   - 컴포넌트 분리
```

### 3단계: 상호작용 디자인

```
각 화면의 상태 변화를 프로토타이핑:

예: 데이터 업로드
- Frame 1: UploadDefault
- Frame 2: UploadDragOver (드래그 시)
- Frame 3: UploadSuccess (드롭 후)

Figma의 "Prototype" 탭에서 연결
```

## 💡 고급 팁

### 1. 브라우저 개발자 도구 활용

```
F12 → Elements 탭
- 요소 선택 (Cmd/Ctrl + Shift + C)
- Computed 탭에서 정확한 크기 확인
- Styles 탭에서 색상 코드 복사
```

### 2. 특정 상태만 보기

갤러리 대신 특정 화면만 보고 싶다면:

```tsx
// App.tsx
import { ChatStreaming } from "./components/figma-screens";

function App() {
  return (
    <div className="dark">
      {" "}
      {/* 다크 모드 적용 */}
      <ChatStreaming />
    </div>
  );
}
```

### 3. 여러 해상도로 한 번에 캡처

개발자 도구 디바이스 모드에서:

- iPhone SE: 375×667
- iPad: 768×1024
- Desktop: 1920×1080

각각 스크린샷 → 피그마의 다른 프레임에 배치

### 4. 애니메이션 표현

피그마에서는 정적이므로 여러 프레임으로 표현:

- 펄스 효과: 크기 다른 2개 프레임
- 호버 효과: 기본 + 호버 상태 2개 프레임
- 스트리밍: 커서 위치 다른 3개 프레임

## 🔧 문제 해결

**Q: 화면이 제대로 표시되지 않아요**

```
A: 브라우저 캐시를 지우고 다시 로드하세요 (Cmd/Ctrl + Shift + R)
```

**Q: 다크 모드가 작동하지 않아요**

```
A: 갤러리 모드에서 우측 상단 버튼을 클릭하거나
   컴포넌트를 <div className="dark">로 감싸세요
```

**Q: 특정 화면만 보고 싶어요**

```
A: index.tsx에서 원하는 컴포넌트만 임포트하여 사용하세요
```

**Q: 스크롤이 있는 긴 페이지를 캡처하고 싶어요**

```
A: Chrome 확장 프로그램 "Full Page Screen Capture" 사용
```
